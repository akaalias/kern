# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## What this is

Kern is a macOS text editor ŌĆö Emacs keybindings/editing model with an iA Writer look. A thin SwiftUI shell launches an editor written in C (SDL2 + OpenGL, `stb_truetype` for fonts). There is **no UI toolkit**: the editor draws immediately through a 12-function renderer interface. (The repo/folder is named `microui` for historical reasons ŌĆö that dependency was removed; geometry types now come from `Editor/gfx.h`.)

## Development process (non-negotiable)

Every feature and fix follows this exact order ŌĆö no step is skippable:

1. **Red** ŌĆö write the failing headless test FIRST (`tests/unit_*.c`, plus any new `platform_stub` controls it needs) and run `make test` to watch it fail.
2. **Green** ŌĆö implement until the suite passes. Put logic where tests can reach it: pure C behind the seams, even when the caller is Swift (see `kern_feed_skip_post`/`kern_feed_quote_text` ŌĆö feed logic in C, called from the Swift formatter via the bridging header).
3. **Build** ŌĆö `xcodebuild` (command in *Commands* below).
4. **Launch** ŌĆö open the Debug app and confirm a **live pid** (`pgrep -x Kern`).
5. **User verifies** ŌĆö tell the user what to test and **stop**. No commit, no push, no release until they explicitly confirm it works.
6. **Ship** ŌĆö commit to `main`, push (the pre-push gate runs), `scripts/release.sh`, commit the pbxproj version bump.

External API details (endpoint paths, params, scopes) are verified against the provider's docs before use ŌĆö never written from memory.

## Features

**Keep this list current ŌĆö add an entry whenever a user-facing feature lands.** It exists so a session can answer "do we have X?" without spelunking. Keybindings are tabulated in `README.md`.

- **Daily notes** ŌĆö `Cmd-Shift-T` jumps to (or creates) today's note at any time; the buffer auto-saves every few seconds while modified ŌĆö **non-blocking** (`autosave_async`): the buffer is snapshotted on the main thread and the disk write runs on a detached background thread so typing never stalls (one save at a time via `g_saving`; optimistic dirty-clear; synchronous + deterministic under `KERN_HEADLESS_TEST`). **Launch routing** (`editor_main`, `textview.c`): if today's `YYYY-MM-DD.md` doesn't exist yet it's created (seeded with a date heading) and, if a previous session had a file open, a backlink to it is dropped in; if today's note already exists, the last session's file is reopened instead.
- **Session persistence** ŌĆö the open file and view toggles (typewriter / syntax mask / style mask) are saved to `.kern_session` in the documents dir on each autosave tick and on quit (`session_save`), and restored on launch (`session_load`) ŌĆö so a relaunch resumes the same file and modes. The startup pin seeds `content_h` + the mono flag before `nav_ensure_cursor_visible` so the restored caret lands at the right strike height. App-only (sandboxed-path-guarded; no-op headless).
- **Per-file cursor memory** ŌĆö every file remembers its own caret position, persisted to `.kern_positions` (`filepos_*` in `textview.c`; a `{path,line,col}` table). The position is stashed on leaving a file, every autosave tick, and on quit, and restored whenever the file is reopened ŌĆö hooked into the central `open_or_create_file` so it covers wikilink-follow, buffer switch (`C-x b`), the daily-note jump, and the launch routing.
- **Emacs editing model** ŌĆö movement, kill/yank ring, operation-based grouped undo (`C-/`), open-line, transpose, word case ops (`editing.c` / `undo.c`, dispatched via `commands.c`). `C-n`/`C-p` and the up/down arrows move by **visual** row (wrapped paragraphs step one screen line at a time), keeping a pixel goal column ŌĆö `nav_visual_move` in `navigation.c`.
- **Mark & region** ŌĆö `C-Space` mark, cut/copy/yank synced with the macOS clipboard, select-whole-buffer, exchange point/mark. **Backspace/Delete over an active selection delete the whole region** (without touching the kill ring) ŌĆö `ed_delete_region` in `editing.c`, called from `cmd_backspace`/`cmd_delete` (falls back to the single-char delete when there's no region).
- **Wrap selection** ŌĆö with a region marked, typing a markdown emphasis/code/highlight char (`*`, `_`, `` ` ``, `=`) **surrounds** the region with that marker on both ends instead of replacing it, and leaves the region active over the inner text ŌĆö so typing `*` twice turns a selection into `**bold**`, and `=` twice into `==highlight==`. Core is `ed_wrap_region(ed, open, close)` in `editing.c` (insert close-then-open as one undo group, re-anchor mark/cursor onto the inner text preserving orientation; no-op on an empty/markless region, returning 0 so the caller falls back to a normal insert). Triggered from the `SDL_TEXTINPUT` path in `textview.c` (gated on `mark_active` + a single-byte wrap char). Headless-tested in `tests/unit_editing.c` (`ed_wrap_region` directly) and `tests/unit_textview.c` (the real typed-`**` event flow).
- **Search** ŌĆö incremental `C-s`/`C-r` with match highlights (`navigation.c`).
- **Live markdown** ŌĆö headings / bold (`**`) / italic (`_`) / highlight (`==`) / inline code (`` ` ``) / **underline (`++text++`)** styled as you type (`md_render.c`; the span kinds are the `SP_*` enum, underline is `SP_UNDERLINE` ŌĆö drawn as a solid baseline rule in the text's own color, no width change so caret/click parity is automatic). **`> ` blockquote lines render in italic** (classic quote styling): `md_is_blockquote` gives the line an italic *base* style in both `md_draw_text` and `md_x_to_col` (so click/caret parity holds even when italic measures wider ŌĆö tested with `stub_set_style_extra`), inline spans still override per glyph (bold stays bold), and the `> ` marker dims like other delimiters. **`==highlight==` is drawn as a yellow over+under rule** (a line along the top edge `y+1` and the bottom edge `y+font_h-1`), not a full-height fill; its bottom rule is one row below the `++` baseline underline (`font_h*0.92`) so a run that is both highlighted and underlined shows two distinct lines rather than one overwriting the other. (The span model is flat ŌĆö `md_scan_spans` resumes after each span, so a single glyph is only ever one `SP_*` kind; the distinct rows keep the two rules from colliding by design.) The wrap-selection chars include `+` so a marked region + `+`├Ś2 becomes `++underline++`. **`Cmd-Shift-H`** / **`Cmd-Shift-U`** toggle a `==highlight==` / `++underline++` around the **sentence the caret is in** ŌĆö no selection needed: `ed_toggle_sentence_highlight` / `ed_toggle_sentence_underline` (both thin wrappers over `ed_toggle_sentence_wrap(ed, '='|'+')` in `editing.c`) find the sentence bounds on the caret's logical line (previous `.!?` ŌåÆ next `.!?`, with surrounding spaces/markers trimmed **both** ends so a no-terminator sentence toggles off cleanly), and either wrap it or, if already wrapped in that marker, remove it ŌĆö one undo group, no-op in the read-only Context section. Detection looks at the markers immediately hugging the sentence, so highlight and underline nest and each toggles independently. Headless-tested in `tests/unit_editing.c` (wrap/unwrap/undo/neighbors/no-terminator/underline+nesting) and the key routing in `tests/unit_textview.c`.
- **Notes & wikilinks** ŌĆö `[[` autocomplete, `Cmd-Enter` follows the link, `Cmd-Shift-ŌåÉ/ŌåÆ` history; `Cmd-Shift-N` extracts the selection into a new linked note (`textview.c`, `recent.c`). The `[[` autocomplete (`buf_list_matches` in `buffer.c`, capped at `WL_MAX`=5) matches **note files only** (`.md`/`.markdown`/`.txt`), **excludes the current file** (MRU index 0), and works two ways: a **bare `[[`** lists the most-recently-opened notes (MRU order, recent doc on top ŌĆö `recency_rank` consults `recent.c`); **typing** filters to notes whose name *contains* the query as a case-insensitive **substring** (`match_score`/`find_ci` ŌĆö "Hello" ŌåÆ "Hello World"/"My Hello", deliberately NOT a loose subsequence that would match "histories"), ranked prefix > word-start > mid-word, then recency, then alphabetically. (`buffer.c` links `recent.c` ŌĆö added to the perf harness too.)
- **Context section** ŌĆö an auto-generated, **read-only** "Context" block appended below every document on open, listing related notes: **backlinks** (other notes whose text contains a `[[ThisNote]]` wikilink), notes **created the same calendar day** (by file birth time), and **"Opened after"** ŌĆö the notes that were open right before this one was opened (see the *Opened-after history* entry below). A **dotted divider with a centered "Context" label** (drawn in `do_render` at `readonly_from + 1`, with blank spacing lines above/below) separates it from the editable page ŌĆö there's no literal `## Context` heading line. The **caret turns warm amber** (the complement of its default cyan) whenever it's inside the section, signalling read-only. `M->`/`Cmd-S-.` (bottom of document) lands the caret at the **true end** ŌĆö the last line of the Context section (read-only but navigable); **select-whole-buffer** (`C-x h`) still stops at the content boundary (`buf_content_line_count`) so a select-all + cut can't touch the section. Real buffer lines, but static: `EditorState.readonly_from` marks where the section begins, the `ed_*` editing primitives refuse to mutate any line `>= readonly_from` (so the caret can still move in and follow wikilinks, but nothing's editable), and save/serialize stop at `buf_content_line_count()` so the section never hits disk or the X-publish payload. The boundary is kept in sync as content above it changes by `buf_insert_line_at`/`buf_delete_line_at`. Generated in `context_refresh()` (`textview.c`, app-only ŌĆö scans the documents dir + creation times; no-op under `KERN_HEADLESS_TEST`, where the read-only/save machinery is unit-tested directly via `readonly_from`). Margin-note footnote definitions insert at `readonly_from` (above the section) so they stay editable and saved.
- **Opened-after history** ŌĆö every note remembers the files that were open *right before* it was opened, surfaced as the **"Opened after"** list in the Context section. The transition is recorded whenever the open file changes ŌĆö `C-x C-f` find-file, `C-x C-b` buffer switch, a `Cmd-Enter` wikilink follow (and back/forward history), and the `Cmd-Shift-T` daily-note jump all funnel through `open_or_create_file`/`cmd_open_daily_note`, which call `opened_after_record(new, prev)` before `context_refresh()`. Kept as a per-file table of predecessor **basenames** (`g_opened_after` in `textview.c`, MRU-ordered, deduped, capped at `OA_PRED_MAX`=24), so a repeat open lifts the predecessor to the front rather than duplicating it. Persisted to `.kern_opened_after` in the documents dir (`opened_after_save`/`opened_after_load`, app-only like `.kern_positions`; tab-separated `predbase\tfullpath`, oldest-first so a reload rebuilds the same MRU order). `opened_after_list` lists the predecessors verbatim (**no on-disk existence filter** ŌĆö a note opened via `C-x C-f` may not be saved to disk yet, e.g. a blank new note that lives only in memory until autosave, but it was genuinely opened so it belongs in the list; a `[[wikilink]]` to a not-yet-existing note just creates it on follow, as everywhere). The record/table logic compiles into both builds and is headless-tested in `tests/unit_textview.c` (the `tv_test_opened_after` seam) ŌĆö predecessor recording, most-recent-first order, and dedup-by-lift ŌĆö while the rendering is app-only.
- **Files** ŌĆö find/open with inline completion, save, save-as, recent-buffer switch (MRU); all paths sandboxed under the app container.
- **View** ŌĆö font size `Ōīś=`/`Ōīś-`, recenter `C-l`, goto-line `M-g`, list indent/outdent (`Tab`/`Shift-Tab`).
- **Typewriter mode** ŌĆö `C-x t` toggles; the active line pins at the golden ratio (~38% down the page) and the buffer scrolls under the cursor. Virtual whitespace below EOF lets the last line still pin. Shares `nav_pin_cursor` with `C-l` recenter; flag is `ViewState.typewriter_mode` (`navigation.c` / `commands.c`). The caret is **also pinned horizontally** at the page-column center: the whole text pane is translated by `-ViewState.scroll_x` (eased with `HSCROLL_EASE`) so the caret holds a fixed screen x while the page slides under it; geometry stays in natural coords ŌĆö only the render/highlight origins subtract `scroll_x` and `nav_click_to_cursor` adds it back. A **hard right margin** blocks typing past the writable width (`nav_at_right_margin`) so lines don't auto-wrap onto a new row ŌĆö only Enter starts a new line. The page is dressed with guides (dotted writing-area edges + solid page margins, tracking `scroll_x`) ŌĆö the same fixed framed page as normal mode (see **Page furniture**; a closed rectangle with top/bottom edges) ŌĆö and two **frosted-plastic guards** flanking the strike point plus a **third panel** covering everything below them to the content bottom (`draw_typewriter_fog` in `textview.c`): each region real-gaussian-blurs the text under it (`r_blur_rect` ŌĆö framebuffer copy + weighted-additive kernel, scissor-clipped) plus a tint; the two guards are additionally stencil-masked to a rounded shape (`r_clip_mask_*`) and pinned at the fixed strike line so they don't bounce as the page glides. `r_blur_rect`/`r_clip_mask_*` are real-GL additions below the renderer seam (app-only; headless stubs in `tests/stub_renderer.c`). **Virtual-space carriage:** the caret can sit *past* end-of-text, holding the horizontal strike point over blank page like a real carriage (`ViewState.virtual_col` = columns past EOL; live only while the caret is still where the move that set it left it ŌĆö `tv_virtual_cols`). A vertical move (`nav_visual_move`) onto a shorter line floats the caret to the goal column instead of snapping to EOL; the horizontal pin is the caret's real x **plus** the float (`tv_virtual_px`), so vertical moves hold the page's horizontal place and arrow floats glide it. **Left/right arrows are horizontal-only, up/down vertical-only** (`tv_horizontal_move`): right past EOL floats (capped at the right margin), left retracts then walks the text, neither jumps lines (shift-select / word-jump keep normal text movement). Typing while floated pads the gap with spaces as one undoable insert. Typewriter mode also **hides the scrollbar** and renders **everything in mono** (`iAWriterMonoS`): `md_set_force_mono` makes `md_draw_text`/`md_x_to_col` draw every glyph in `FONT_MONO`, and `md_wrap_font_style()` (FONT_MONO when forced, else FONT_REGULAR) is what `navigation.c`'s `body_width`/`nav_get_wrap_breaks`/`nav_at_right_margin` measure with, so wrap/click/render stay in lockstep; the `C-x t` toggle flips the flag and invalidates all wraps to reflow in the new font.
- **Syntax highlighting** ŌĆö `C-x y` toggles part-of-speech highlighting. Unlike iA Writer's five-hue rainbow, the palette is **value, not hue** (Tufte): a luminance ramp by grammatical importance ŌĆö verb (brightest) > noun > adjective > adverb >> function words (determiner/preposition/pronoun/conjunction/particle, dimmed so the grammatical glue recedes). What you read is the prose's content rhythm, figure vs ground. Tagging is behind a seam (`pos_tagger.h`): the app backs it with `NSLinguisticTagger` lexical-class scheme (`pos_tagger_nl.m` ŌĆö same engine iA uses), tests with a fixture lexicon (`tests/pos_tagger_fake.c`). Per-line spans are lazily cached on the `Line` (`pos_spans`, invalidated by `line_dirty` like `md_spans`); `pos_render.c` owns the cache, the value palette, and a determinerŌåÆnoun post-correction (a gerund right after "a/the" ŌĆö "a heading" ŌĆö is retagged from verb to noun). `md_draw_text` applies the per-span value, layering over markdown so bold/italic keep their weight. Bit-per-class mask is `ViewState.syntax_mask` (0 = off). **Highlight is an isolate, not a hide:** when the mask is non-zero, a shown class takes its ramp value and *everything else* ŌĆö toggled-off classes and untagged/`POS_OTHER` text ŌĆö drops to a muted ground (`pos_mute_color()`, one step below the function-word value) so the shown classes pop. The mute is applied in `md_draw_text` (the `else` of the `pos_color_at` branch), not in `pos_color_at` (whose `1=shown / 0=not` contract is unchanged). Per-type toggles live in the **View** menu: Verbs / Nouns / Adjectives / Adverbs / **Function Words** (one item for all five function-word classes, since they share the palette value), each flipping one bit of `syntax_mask` via a `kern_toggle_*` bridge; the master "Syntax Highlighting" (and `C-x y`) flips all bits. **Type-in-progress hold + fade-in:** the word currently being typed would re-tag (and so re-color) letter-by-letter as it grows ("comp"ŌåÆ"compute"ŌåÆ"computer"), which flickers. So the word under the caret is held at the base ("max") color ŌĆö neither colored nor muted ŌĆö until it's finished, then fades from the base color up to its real POS color over `POS_FADE_MS` (5s, smoothstep ŌĆö "ink drying"). A word is "in progress" only because an **edit** put the caret in it (`buf_edit_seq` change in `pos_anim_track`, `textview.c`); it finishes when the caret leaves it ŌĆö a space, punctuation, an arrow, a click ŌĆö so merely *navigating* onto already-colored text never disturbs its color. The timing/animation state lives in `pos_render.c` (`pos_set_wip` / `pos_fade_begin` / `pos_resolve_color`, a small fade ring keyed by `(Line*, byte range, t0)`) **not on the `Line`**, since `line_dirty` rebuilds the per-line span cache on every keystroke; `md_draw_text` calls `pos_resolve_color` in place of the raw `pos_color_at`/`pos_mute_color` choice. The fade drives the event loop at ~60fps via a `g_pos_animating` flag (ORed into `SDL_WaitEventTimeout`, beside the scroll/focus flags). Pointer-compared, never dereferenced (stale-pointer-safe, like `sub_render`'s reveal range). Headless-tested in `tests/unit_pos.c` (`pos_word_bounds`, the in-progress hold, the timed lerp).
- **Style check** ŌĆö `C-x s` toggles iA-Writer-style strike-through of cuttable text. A third per-line span layer (`style_check.c` ŌåÆ `Line.style_spans`, invalidated by `line_dirty` like `md_spans`/`pos_spans`). Each category gets a distinct line texture (`StyleDecor`), drawn in `md_draw_text` as dashes with the `wave` wobble (one `r_draw_rect` per dash): **filler** ŌåÆ greyed strike through the x-height ("delete"); **redundancy** ŌåÆ a wavy underline on the baseline, on only its cuttable word(s) (`cut_from`/`cut_len` per pattern, so "a╠Čd╠Čd╠Če╠Čd╠Č bonus" keeps "bonus"); **clich├®** ŌåÆ a straight dotted underline ("rewrite", text kept readable). Pure C ŌĆö curated word/phrase lists (fillers, redundancies, clich├®s/generalizations) in `k_patterns[]`, whole-word/longest-match, no tagger or renderer dependency, fully headless-tested (`tests/unit_style.c`). Categories are `StyleCategory` (`STYLE_FILLER`/`STYLE_REDUNDANCY`/`STYLE_CLICHE`); bit-per-category mask is `ViewState.style_mask` (0 = off). Detection is precision-first (context-free entries only).
- **Symbol substitution** ŌĆö `C-x l` toggles display-only textŌåÆsymbol rendering ("ligatures"): the buffer keeps the literal bytes but the renderer draws a single glyph in their place ŌĆö `->`ŌåÆŌåÆ, `<-`/`=>`/`<=>`, `!=`/`<=`/`>=`/`~=`, `--`ŌåÆŌĆö (em), a digit-`-`-digit en dash, `...`ŌåÆŌĆ”, `+-`ŌåÆ┬▒, `(c)`/`(r)`/`(tm)`, directional smart quotes `"`/`'` (the close form also gives the right apostrophe in "don't"), **Greek by name** (`lambda`ŌåÆ╬╗, `Sigma`ŌåÆ╬Ż, ŌĆ” ŌĆö whole-word, case-sensitive), and a few **math operator words** (`forall`ŌåÆŌłĆ, `exists`ŌåÆŌłā, `nabla`ŌåÆŌłć, `infinity`ŌåÆŌł×, `sqrt`ŌåÆŌłÜ). Greek/math are `word` entries in `k_fixed[]`: matched only as whole words (bounded by non-`[A-Za-z0-9_]`) so `lambdas`/`lambda_x` stay literal. English-colliding triggers (`in`, `sum`, `and`, `or`, `partial`, the short Greek `eta`/`nu`/`xi`/`chi`) are **deliberately excluded** ŌĆö no clean prose-vs-math disambiguation, and reveal-on-contact can't rescue a false positive you didn't notice. **Font-glyph fallback:** the bundled body font has partial Unicode coverage (it lacks e.g. ŌćÆ Ōćö ŌłĆ Ōłā Ōłć ╬®), so `sub_render`'s `emit` skips any substitution whose glyph `r_has_glyph` (a renderer-seam call ŌåÆ `stbtt_FindGlyphIndex`) reports the font can't draw ŌĆö the source renders as literal text instead of a tofu box. This makes `sub_render.c` renderer-dependent (was pure C); the test stub's `r_has_glyph` returns present-unless-`stub_set_glyph_missing`, so the table is still exercised headlessly. A **fourth** per-line span layer (`sub_render.c` ŌåÆ `Line.sub_spans`, invalidated by `line_dirty` like the other three) ŌĆö `SubSpan{start,len,category,glyph}`, scanned longest-match-first from a curated table (`k_fixed[]`) plus a little context logic for quotes/en-dash. **Reveal-on-contact**: a token is drawn collapsed (the glyph) unless the caret touches it `[start, start+len]`, in which case the render/measure walks draw the literal source bytes so it stays editable. The caret's line+column is published each frame via `sub_set_caret` (and at event time by the click/vertical-move paths); `sub_reveal_col(l)` returns the caret column for the caret's line (compared by pointer, never dereferenced ŌĆö a stale value after a line-array realloc is harmless), -1 elsewhere. The geometry-critical invariant is that the two render/measure walks ŌĆö `md_draw_text` (render + caret-x) and `md_x_to_col` (click + vertical move) ŌĆö agree on every token's width *and* its reveal state; both read `sub_line_spans` + the active mask (`sub_active_mask`, set per frame from `ViewState.sub_mask`) + `sub_reveal_col`. **Wrap (`nav_get_wrap_breaks`) deliberately measures the literal text, ignoring substitution** ŌĆö a collapsed glyph is never wider than its source, so a literal-width wrap point always fits when drawn collapsed, and revealing a token redraws exactly the width wrap already reserved ŌåÆ no reflow as the caret moves, and the wrap cache stays valid with no toggle-time invalidation. Pure C, fully headless-tested incl. drawŌåöclick parity, reveal-on-contact, selection-range reveal, and wrap substitution-independence (`tests/unit_sub.c`). **Reveal is a range, not a point:** the active selection's extent on a line (unioned with the caret point) reveals every token it overlaps, so a selected symbol stays expanded instead of flip-flopping as the caret passes ŌĆö `nav_sub_reveal_for_line(ed, ln)` computes it (caret Ōł¬ `buf_region_ordered` extent) and the render/measure passes call it **per visual row**. Two ordering rules keep the marked region from looking wonky around symbols: (1) `do_render` sets `sub_set_mask` *before* `process_frame`, so the selection/search highlights (`md_col_x`) measure the same collapsed/revealed widths the text pass draws; (2) every `md_draw_text`/`md_col_x`/`md_x_to_col` site sets the per-line reveal first. Categories are `SubCategory` (`SUB_PUNCT`/`SUB_ARROW`/`SUB_GREEK`/`SUB_MATH`/**`SUB_LIGATURE`**); mask is `ViewState.sub_mask`, defaulted to `SUB_MASK_ALL` in `editor_main` (**on by default**; `tv_test_reset` leaves it 0 so the suite's other geometry tests are unaffected, and `unit_sub.c` sets the mask explicitly). Not yet in the View menu (keyboard-only for now). **Typographic f-ligatures** (`ff` `fi` `fl` `ffi` `ffl` ŌåÆ ’¼Ć ’¼ü ’¼é ’¼ā ’¼ä) are `SUB_LIGATURE` entries in `k_fixed[]` ŌĆö **not** whole-word (they sit mid-word in "offer"/"file"), longest-match-first so ffi/ffl beat ff. They render only when the bundled font supplies the Alphabetic-Presentation-Forms glyph (else the `r_has_glyph` fallback leaves the source literal).
- **Page furniture** ŌĆö the page borders (dotted writing-area edges, two-line note gutters, solid page-margin rails, top/bottom edges) and **margin notes** render in **both** typewriter and normal (non-typewriter) editing mode; in normal mode this is gated on the window being wide enough to hold the note strip (`nav_page_margin() >= tv_margin_pad()`), otherwise notes fall back to bottom-of-document footnotes. The borders live in `draw_page_furniture` (split out of the typewriter fog). **One drawing for all modes:** a fixed **framed page** ŌĆö a closed rectangle filling the writing area (top of content down to just above the status bar), with the vertical rails clipped to the frame and solid **top/bottom edges** closing it off (no scroll-tracking paper edges, no full-window rails). Normal mode gets a top page margin via a negative `min_scroll` (reusing typewriter's whitespace-above mechanism ŌĆö no nav geometry changes) and a bottom margin by trimming `content_h`. Default font size is 24pt.
- **Page-border toggle** ŌĆö **`C-x p`** (`page_furniture_hidden` in `ViewState`) hides/shows *all* page furniture ŌĆö rails, dotted edges, and note gutters ŌĆö in **both** normal and typewriter mode (gates the `draw_page_furniture()` calls in `do_render`; margin notes and the typewriter frost still draw). Persisted across launches in `.kern_session` (the `noborders` line, alongside `typewriter`/`syntax`/`style`). Key routing is in `handle_cx_prefix_key`; headless-tested in `tests/unit_textview.c` (`test_cx_p_toggles_page_furniture`).
- **Margin notes / footnotes** ŌĆö `Cmd-Shift-M` (no selection needed; typewriter mode or a wide-enough normal-mode window) opens a keyboard-first margin-note input: the hitzone slides to the right page margin (document frosts behind it, single caret), you type, and **Enter** inserts a `[^id]` footnote marker at the write-head and appends a `[^id]: note` definition at the document bottom (**Esc** cancels). Stored as a **standard markdown footnote**, so in normal mode it's just a footnote line at the bottom; in typewriter mode `draw_margin_notes` (`textview.c`) renders each definition in the right-margin strip aligned to its reference's *visual* row (mono, **80% of the body font size and line-height** (`MN_SCALE`), word-wrapped ŌĆö via `r_set_font_scale`, a quad-level draw/measure scale on the renderer seam that reuses the existing atlas, so unlike `r_set_font_size` it's safe per frame). State is `mn_*` file-statics; id is base36 of the clock+seq. **Overlap handling:** notes flow down the right margin; when one would collide with the note above it, it **spills to the left margin** instead. The right/left side is cached per id (`g_fn_sides`) and only recomputed (in scroll-independent document space, `recompute_footnote_sides`) when a footnote id appears that isn't cached yet ŌĆö i.e. when a new note is written or a new doc loads ŌĆö so scrolling never reshuffles sides. While a new note is being typed, its live input reserves a growing extent and saved notes within it spill left early. **Two render passes:** saved notes draw *before* `draw_typewriter_fog` (so the frost blurs them with the page), the live input draws *after* (sharp, in the hitzone moved to the margin). Notes live in a padded, two-dotted-line **gutter** in each page margin (`tv_note_col`/`tv_note_gutter`, strip width `tv_margin_pad`); `Cmd-Shift-M` scrolls the page so the right note column lands in the fixed centre hitzone (the page comes to the note). A note is drawn whenever **any part** of it is on screen ŌĆö `draw_margin_notes` walks the buffer tracking the running visual-row index to get each reference's y even when its marker row is above the viewport ŌĆö so notes don't vanish early on scroll. **Gotcha:** the margin renderer must never call `r_set_font_size` (a size change rebuilds all four font atlases ŌĆö doing it per frame lags the whole app); the smaller size comes from `r_set_font_scale`, applied **only inside `draw_note_wrapped`** (set `MN_SCALE` ŌåÆ `draw_wrapped` ŌåÆ restore 1.0). Never leave the scale set while nav geometry runs ŌĆö `nav_line_height`/`r_get_text_height` and `nav_get_wrap_breaks` are scale-sensitive, so an ambient scale shrinks line heights (mis-anchoring `refY`) and corrupts the body wrap caches. The live-input caret matches the body caret's shape (3px ├Ś `MN_SCALE`-scaled text-height). Headless-tested in `tests/unit_textview.c` (the `Cmd-Shift-M` authoring flow: marker + appended definition, Esc-cancel, no-room no-op on a narrow window, and normal-mode authoring on a wide window via `stub_set_metrics`).

## How to verify a change (important)

The test build compiles the core **plus** `Editor/textview.c` ŌĆö built with `-DKERN_HEADLESS_TEST`, which `#ifdef`s out only `editor_main`/`resize_event_watcher` (the real-SDL init + blocking loop) and exposes the `tv_test_*` seam. So:
- Changes to `buffer/editing/undo/navigation/md_render/recent/commands.c` ŌåÆ verify with `make test` (and `make coverage`).
- Changes to `textview.c`'s **event/dispatch + modal layer** (the `editor_handle_event` switch, prefix chords, wikilink nav, minibuffer, isearch, autosave in `editor_tick`) ŌåÆ covered by `tests/unit_textview.c`, which pumps synthetic `SDL_Event`s through `editor_handle_event` against the stub renderer + `tests/platform_stub.c`. - Changes to the parts still **not** compiled in tests ŌĆö `editor_main` itself, the real-SDL render/init path, AppKit chrome, anything Swift ŌĆö verify by building the app with `xcodebuild` and launching it. Confirm the app actually has a live pid before claiming it works.

## Architecture

**The renderer seam.** `Platform/renderer.h` is a 12-function interface (`r_draw_rect`, `r_draw_text`, `r_get_text_width`, `r_set_font_style`, `r_get_size`, ŌĆ”). The app implements it with SDL/GL (`Platform/renderer.c`); the tests implement it with a deterministic capture stub (`tests/stub_renderer.{c,h}`, 10├Ś20px glyphs, 800├Ś600 window, records every draw op). Any `Editor/` code that needs metrics or drawing goes through this seam, which is what makes layout testable headlessly.

**State lives in two structs.** `EditorState` (document model ŌĆö buffer, cursor, mark, kill ring, undo; no SDL) and `ViewState` (UI ŌĆö scroll, fonts, search/minibuffer/prefix flags, status). Defined in `Editor/editor_types.h`. **Every core function takes explicit `EditorState*`/`ViewState*` pointers.** The only globals are `g_ed`/`g_vs` singletons inside `textview.c`.

**Two tiers of `Editor/` code:**
- *Pure core* ŌĆö `buffer.c` (lines, file I/O, kill buffer, mark/region, sandbox paths), `editing.c` (all `ed_*` ops), `undo.c` (operation-based, grouped). No renderer dependency.
- *Renderer-dependent but still headless-testable* ŌĆö `navigation.c` (wrap, cursor/visual mapping, click-to-cursor, vertical movement, search), `md_render.c` (inline markdown spans + drawing), `commands.c` (the command table + `kern_dispatch_key`), `recent.c` (MRU). These depend only on `renderer.h` (plus `navigation.c` ŌåÆ `md_render.h`, since wrap/click/movement geometry must mirror the render ŌĆö see the gotcha below), so they compile into *both* the app and the test binary. (`navigation.c`'s link graph now pulls `md_render.c` + `pos_render.c` + `style_check.c`; the perf harness in `tests/Makefile` links them too.)
- *Partly headless-testable shell* ŌĆö `textview.c` (~1,400 lines): its per-event work (`editor_handle_event`) and per-frame work (`editor_tick`) are extracted from `editor_main` so the modal/dispatch layer (minibuffer, wikilinks, buffer-switch, prefix chords, isearch, autosave) is driven in `tests/unit_textview.c` via the `tv_test_*` seam (`Editor/editor_loop.h`). The remainder ŌĆö `editor_main`'s real-SDL init + blocking loop and the GL render path ŌĆö stays app-only.

**Command dispatch.** `commands.c` owns the single binding table `g_commands[]` + `kern_dispatch_key(ed, vs, kmod, sym)`. `textview.c`'s keydown handler calls `kern_dispatch_key` first; only things needing local UI state are handled outside it: `M-g` (minibuffer), the `C-x ŌĆ”` and `ESC`/meta prefix chords (`handle_*_prefix_key`), and the arrow keys (which delegate to the `commands.c` movement functions, layering shift-select + word-jump). Feature tests drive `kern_dispatch_key` directly in `tests/unit_commands.c`.

**Lazy per-line caches.** `Line` carries `wrap_count` (cached visual-row count, `-1` = dirty) and `md_spans`/`md_span_count` (cached inline-span map, `-1` = not computed). `line_dirty()` invalidates both on edit; `buf_invalidate_all_wraps()` clears wraps on a page-width or font-size change. `nav_maybe_reflow` skips the full re-wrap when a resize doesn't change page width.

**Determinism seams.** Time goes through `Editor/clock.h` (`kern_now_ms()`; app uses `clock_sdl.c`, tests use a settable `clock_fake.c`). Clipboard goes through `Editor/clipboard.h` (app `clipboard_sdl.c`, tests `clipboard_fake.c`). Use these, not SDL directly, in `Editor/` code.

## Conventions / gotchas

- **Always measure wrap and column math in `FONT_REGULAR`.** A frame ends with the status bar's `FONT_MONO` active; if an event-time wrap recompute measures in mono, it caches wrong wrap counts ŌåÆ phantom rows and mis-placed click-to-cursor. `navigation.c`'s `body_width()` and the wrap functions force `FONT_REGULAR` regardless of ambient style ŌĆö preserve this in any wrap/metric change.
- **Navigation geometry must mirror what `do_render` draws.** Each visual row is drawn at `nav_page_margin() + md_row_indent(line, row_start)` (a list item's continuation rows hang under the item text) with **per-span font widths** (bold/italic/mono differ from `FONT_REGULAR`), and a heading's `### ` may hang in the left margin. Wrap-breaking, click-to-cursor, and the caret-x all go through that same origin + `md_x_to_col`/`md_col_x` (per-span aware) + `nav_heading_markers_hang` ŌĆö otherwise clicks land offset on styled/list lines and wraps drift past the right edge. Keep these three in lockstep with the render pass; `md_row_indent` is the single source of the indent formula.
- **Vertical movement is by *visual* row, not logical line.** `cmd_next_line`/`cmd_previous_line` (and the up/down arrows, which route through them) call `nav_visual_move`, which steps one screen row and keeps a *pixel* goal column in `ViewState.goal_x/goal_line/goal_col` (reused only while the caret is still where the last vertical move left it). Moving a whole logical line on `C-n` inside a wrapped paragraph is the bug this replaced. Up past the top / down past the bottom land at buffer start/end.
- **Columns are byte-indexed**, not codepoint-indexed (UTF-8 inserts advance the cursor by byte length).
- **File paths are sandboxed** via `buf_set_documents_dir()` (set from Swift at launch); when unset (CLI/test builds) paths are used verbatim. User paths resolve under the documents dir via `buf_resolve_path`.
- **Test layout:** `tests/ed_fixture.h` provides `ed_load`/`ed_teardown`/`LINE()`; `tests/test.h` provides `CHECK`/`CHECK_IEQ`/`CHECK_SEQ` and `RUN`. Each `unit_*.c` exposes one `suite_*()` listed in `test_main.c`. Snapshots prove render parity; they re-implement the text pass, so a `do_render` text-loop refactor won't be caught by goldens.
- **Coverage reality:** core is ~99% lines / 100% functions; the residual uncovered lines/branches are malloc/IO-failure paths and defensive guards (need fault injection, not more tests). The gated coverage floor (Ōēź85%, `make coverage`) measures `COVCORE` = the core only ŌĆö `textview.c` is **deliberately excluded** so its partly-tested shell doesn't mask the core's saturation. `textview.c` is ~53% lines (event/dispatch + modal handlers tested; uncovered remainder is the `do_render`/`process_frame` GL chrome and the `#ifdef`'d-out `editor_main`); see it with `make coverage-textview`.
- **Pumping events in tests:** drive `editor_handle_event` with synthetic `SDL_Event`s (`tests/unit_textview.c` has `key()`/`textinput()`/`type()` builders). The modal-routing precedence is real ŌĆö don't bypass it by calling `handle_*` directly. The wikilink dropdown (`wl_active`) is recomputed inside `do_render`, so activate it by typing then calling `editor_tick()` before sending the accept/dismiss key. `tv_test_reset()` clears `g_ed`/`g_vs` + all modal file-statics between tests; `platform_stub.c` fakes the SDL input-state calls and the `kern_x_*` bridge. **Keep `editor_main` loop-shape-preserving** (wait ŌåÆ drain via `editor_handle_event` ŌåÆ `editor_tick`) ŌĆö that's what keeps it both snappy and testable.
- **The main thread is parked by `editor_main`.** It runs SDL's blocking event loop and never returns, so the main thread / main actor is stuck for the app's life. Any Swift feature added to the app **must run off the main actor and report UI through the C status bar** (`kern_x_set_status`). `DispatchQueue.main.async`, main-actor `Task {}`, SwiftUI live updates, and `ASWebAuthenticationSession` all silently deadlock. Background threads (`Task.detached`, GCD, `NWListener`, `URLSession`, `NSWorkspace.open`) work fine. The X publishing layer (`App/KernApp.swift`) is the reference pattern.
