Kern

A macOS text editor with Emacs hands, iA Writer looks, and Obsidian links.

Three apps, three loves, three lingering frustrations. Kern is the editor that keeps what each does best and drops the rest. It's a small native thing — a SwiftUI shell over an editor written in C, drawing every glyph itself. No Electron, no plugin archaeology, no toolkit.

The three apps I couldn't fully quit

1Emacs

Fully muscle-memorized, blindingly fast text editing — no second-guessing, ever. But markdown rendering needs extra packages, the typography isn't iA Writer, and link-navigation never feels as smooth as Obsidian.

2iA Writer

Amazingly minimal — I've happily paid for it four times in ten years — with typography I'd defend in a fight. But none of the Emacs keybindings and chords, so editing feels generic, and its wikilinks sit well below the bar Obsidian set. Next to Emacs, it feels bloated.

3Obsidian

Since 1.0, the best on-the-fly linking there is — add a link mid-thought, follow the link under the cursor, jump back, turn a selection into its own linked note. But it's Electron, the Emacs community plugins never quite worked, typography is so-so, and the editing itself is generic and uninspired.

So Kern takes the muscle memory from one, the type from the next, and the linking from the third — and refuses the bloat of all three.

What it does

Emacs editing
Movement, kill/yank ring, grouped undo (C-/), transpose, word-case ops, open-line.
Mark & region
C-Space to mark; cut, copy, and yank synced with the macOS clipboard.
Wikilinks
[[ autocompletes; ⌘↵ follows the link; ⌘⇧←/→ walks history.
Extract to note
⌘⇧N turns the selection into a new note and leaves a link behind.
Live markdown
Headings, bold, and italic styled as you type — no preview pane.
Daily notes
Opens today's note on launch; ⌘⇧T jumps back to it; autosaves while you write.
Search
Incremental C-s/C-r, plus ⌘F with match highlights.
Typewriter mode
C-x t pins the active line at the golden ratio and scrolls the page beneath it.
Typography
iA Writer Quattro and Mono — the very faces rendering this page.
